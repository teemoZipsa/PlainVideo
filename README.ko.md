# PlainVideo

> 영상만 남기는, 테두리 없는 Windows 동영상 플레이어.

PlainVideo는 PlainView의 제품 철학을 동영상으로 확장하는 완전한 오픈소스 프로젝트입니다. 파일을 열면 바로 재생하고, 평소에는 영상 외의 UI를 숨기며, 필요한 순간에만 최소한의 조작부를 보여주는 것을 목표로 합니다.

현재 상태는 **임베디드 재생 검증 단계(pre-alpha)** 입니다. PlainVideo가 소유한 창 안에서 libmpv로 직접 렌더하는 로컬 프로토타입은 있지만, 아직 배포 가능한 플레이어는 아닙니다.

이제 PlainVideo는 대응하는 portable 빌드를 함께 유지하면서, 패키지형 Win32 MSIX로 배포하는 무료 Microsoft Store 앱을 목표로 합니다. 현재 개발용 libmpv 런타임은 재배포 승인이 끝나지 않았으므로 런타임 라이선스·빌드 확정이 첫 Store 출시 게이트입니다. 자세한 순서는 [Microsoft Store 출시 계획](docs/STORE_RELEASE_PLAN.md)에 기록합니다.

## 제품 원칙

- **영상이 화면의 주인공** — 제목 표시줄, 테두리, 고정 도구 모음을 기본 화면에 두지 않습니다.
- **설명 없이도 자연스럽게** — 클릭은 재생/일시정지, 더블 클릭은 전체 화면처럼 이미 익숙한 동작을 우선합니다.
- **필요한 순간에만 UI** — 재생/탐색/볼륨을 조작한 직후에만 짧은 피드백이 나타나고 다시 사라집니다.
- **기능은 넓고 표면은 얇게** — 자막, 오디오 트랙, 영상 정보, 고급 화질 기능은 필요할 때만 단계적으로 드러냅니다.
- **광고·계정·추적 없음** — 로컬 파일 재생을 방해하는 수익화 UI나 텔레메트리를 넣지 않습니다.
- **느린 PC에서도 정직하게** — 무거운 기능은 자동 감지 후 제공하고, 감당할 수 없으면 조용히 기본 재생으로 돌아갑니다.

## 목표 기능

- 파일 열기와 드래그 앤 드롭 즉시 재생
- MP4, MKV, WebM, MOV, AVI, MPEG-TS 계열 등 폭넓은 컨테이너 지원
- H.264, HEVC, AV1, VP9, MPEG-2 계열과 다양한 오디오·자막 트랙 지원
- 하드웨어 디코딩 자동 선택과 안정적인 소프트웨어 폴백
- 같은 폴더의 이전/다음 영상 탐색
- 외부 자막 자동 탐색, 내장 자막·오디오 트랙 전환
- 재생 속도, 화면 비율, 회전, 항상 위, 스크린샷
- HDR 재생과 SDR 디스플레이용 톤 매핑
- 휴대용(portable) 빌드를 포함한 Windows 배포

실제 지원 범위는 확장자 목록만으로 과장하지 않고, 배포 빌드에 포함된 디먹서·디코더를 기준으로 자동 생성한 호환성 표로 공개할 예정입니다.

## 지금 동작하는 것

- 고정된 libmpv 개발 런타임을 불러오는 Rust 네이티브 Windows 셸
- 제목 표시줄과 테두리가 없는 창 안의 in-process render API 재생
- 클릭 재생/일시정지, 더블 클릭 전체 화면, 좌우 방향키 5초 탐색, `Shift`+좌우 30초 탐색, 상하 방향키·마우스 휠 볼륨, `Esc` 복귀
- 평소에는 완전히 숨고 포인터/조작 직후에만 나타나는 반투명 피드백과 간결한 재생 바
- 재생 중 1.6초 뒤 자동으로 숨고 마우스를 움직이면 다시 나타나는 포인터
- 별도 손잡이 없이 이동 커서만 쓰는 상단 전체 너비 56px 비가시 창 이동 영역
- 마우스 활동 때만 나타나는 우상단 `테마 · 항상 위 · 최소화 · 닫기`와 하단 `재생 · 탐색 · 볼륨 슬라이더/음소거 · 자막 · 전체화면` 조작부
- 모니터 간 물리 픽셀 창 위치 저장, DPI 기준 280×240 최소 크기, 네이티브 그림자를 유지한 DWM 무테두리 처리
- 같은 이름의 외부 SRT 자막 자동 탐색, 우클릭 자막·오디오 트랙 선택과 외부 자막 파일 열기
- 한국어/영어가 섞이지 않는 드롭 안내와 `Ctrl+O` 열기 경로
- `PageUp`/`PageDown` 이전·다음 영상, `A` 오디오 트랙, `V` 자막 트랙, `M` 음소거 단축키
- 재생 중 `Tab`으로 좌상단의 간결한 미디어 정보 표시를 켜고 끄는 기능
- `F6`/`Shift+F6`으로 잠깐 나타나는 조작부에 포커스를 옮기고 `Enter` 또는 `Space`로 실행하는 키보드 조작과 포커스 표시
- 실제 파일 드롭 교체와 영상 열기·다시 시도·이전/다음·오디오·자막·닫기를 단계적으로 펼치는 우클릭 메뉴
- 파일 하나의 재생 오류로 앱을 닫지 않고 다시 시도하거나 다른 파일을 열고 놓을 수 있는 복구 가능한 오류 화면
- 파일별 해시를 포함한 재현 가능한 로컬 개발자용 portable 디렉터리

2026-07-19의 첫 로컬 개발자 형식 매트릭스는 해시가 고정된 같은 portable에서 10개 컨테이너·코덱·자막 조합을 모두 통과했습니다. 이는 아직 재배포가 승인되지 않은 더티 작업 트리의 개발 증거이며, 배포판 전체 지원 보증은 아닙니다. 정확한 조합과 한계는 [형식 호환성 장부](docs/FORMAT_COMPATIBILITY_LEDGER.md)에 기록합니다. 임베디드 빌드는 이 개발 PC에서 `nvdec`를 선택했고, 이전 자식 프로세스 Slice 0A에서는 `d3d11va-copy`가 관측됐습니다. 어느 쪽도 다른 GPU의 하드웨어 지원 보증은 아닙니다.

## 로컬 실행

```powershell
.\scripts\bootstrap-mpv.ps1
cargo run --release -- "C:\path\to\video.mkv"
```

결정적 테스트 영상을 다시 만들려면 FFmpeg가 설치된 환경에서 `.\scripts\generate-smoke-media.ps1`을 실행합니다. `.\scripts\build-portable.ps1`은 `.runtime` 아래에 로컬 개발자용 portable 증명을 만들며, 아직 재배포할 수 없습니다. `.\scripts\run-local-validation.ps1`은 빠른 로컬 품질 검증을 한 번에 실행합니다. 이번 범위와 남은 게이트는 [품질 패스 1~6](docs/QUALITY_PASS_1_TO_6.md)에, 세부 기준은 [접근성 기준선](docs/ACCESSIBILITY_BASELINE.md), [형식 호환성 장부](docs/FORMAT_COMPATIBILITY_LEDGER.md), [재생 안정성 계획](docs/PLAYBACK_STABILITY_PLAN.md), [Slice 0B libmpv 증명](docs/SLICE_0B_LIBMPV_PROOF.md), [Windows 창 동작 증명](docs/WINDOW_BEHAVIOR_PROOF.md)에 기록합니다.

## 부드러운 재생과 프레임 더블러

PlainVideo는 서로 다른 두 기능을 구분합니다.

1. **화면 동기화** — 디스플레이 주사율에 맞춰 프레임 타이밍을 부드럽게 만드는 가벼운 모드입니다. 새 장면을 AI로 생성하는 기능은 아닙니다.
2. **프레임 더블러 (실험적)** — RIFE 계열 오픈소스 보간을 이용해 24→48fps, 30→60fps처럼 중간 프레임을 실시간 생성하는 선택 기능입니다.

프레임 더블러는 기본값을 끔으로 두고, 지원되는 Vulkan GPU와 충분한 처리 여유가 확인될 때만 노출하는 방향으로 설계합니다. 탐색·장면 전환·고속 움직임에서 생기는 잔상과 왜곡을 숨기지 않고, 자동 우회와 품질 표시를 제공합니다.

## 기술 방향

- 재생 엔진: [mpv/libmpv](https://github.com/mpv-player/mpv) + FFmpeg
- GPU 렌더링: libmpv render API 기반 하드웨어 디코딩, HDR 및 톤 매핑
- 프레임 보간 후보: [Practical-RIFE](https://github.com/hzwer/Practical-RIFE) 모델 + ncnn/Vulkan 계열 런타임
- UI 셸: 검증된 기준선은 얇은 Rust/Win32 셸이며, 더 높은 수준의 오버레이 도구는 이 네이티브 표면 동작을 보존할 때만 도입합니다.

브라우저의 `<video>` 요소만을 주 재생 엔진으로 쓰는 방식은 광범위한 형식·자막·HDR·하드웨어 제어 목표와 맞지 않아 제외합니다. 자세한 내용은 [아키텍처 노트](docs/ARCHITECTURE.md)와 [제품 정의](docs/PRODUCT.md)를 참고하세요.

## 범위 밖

- 미디어 서버·대형 라이브러리 관리
- 영상 편집 및 변환 도구
- 광고, 추천 피드, 뉴스, 상점

## 이름

`PlainVideo`는 이미지에는 PlainView, 영상에는 PlainVideo라는 제품군 관계와 앱의 용도를 함께 드러내는 이름입니다. Microsoft Store 등 바이너리 공개 배포 전에는 상표와 스토어 이름 사용 가능 여부를 다시 확인합니다.
